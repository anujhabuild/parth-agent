02The toolbelt

A complete tool surface — routed dynamically per turn.

A regex-based router inspects the last four messages and includes only the schemas that are likely needed. Core tools are always loaded; specialty groups (web, mac, memory, skills, MCP) attach on demand.

03The brain

Skills, lessons, memory — and one global library.

Parth doesn't forget. It compounds. Every project keeps its own skills; the global library is reachable from anywhere; lessons accumulate across sessions; MCP plugs in external tools without rebuilding the agent.

Skills
⚒

Skills

Reusable playbooks defined in .skills/<name>/SKILL.md. Lazy-loaded — only their description is injected; full content loads when the task matches.

Global
◉

Global Skills

A Global mode where every skill — from every project — is reachable simultaneously. Build once, use everywhere. No copy-pasting between repos.

Memory
⚇

Memory

Durable facts about you and your projects. Auto-saved when something matters; SQLite-backed at ~/.config/claude-agent/. Survives crashes, restarts, OS updates.

Lessons
✎

Lessons

Post-task notes. After every non-trivial job, Parth writes what it learned. Future sessions search lessons before guessing — compounding intelligence per repo.

MCP · project + global
⇄

MCP Servers

Model Context Protocol with two scopes: per-project (.mcp.json) and global — once added, every project sees those servers. No manual wiring, no per-repo setup. Plug Context7, Postgres, Sentry, Linear, Notion in once.

Sessions
↺

Sessions

Every conversation is a resumable session. Search history, fork a thread, pin context, switch models mid-conversation, reload yesterday's debugging at 9 AM tomorrow.

Hallucination guard
◬

Truth filter

Streaming output passes through a hallucination guard. Specific facts (versions, prices, URLs) are checked against the session's actual fetches before display.

Modes
◐

Modes

Default · Coding · Reverse-engineering. Each swaps in a different system prompt and tool routing bias — same agent, different posture.

08Under the hood

Engineering choices, visible.

Parth is not a wrapper around an SDK. It's a custom streaming tool loop with deliberate tradeoffs at every layer.

  • i.
    Dynamic tool routing. Only the schemas likely needed each turn — regex-scans recent messages. Smaller payloads, faster TTFB.
  • ii.
    Parallel by default. Up to 64 concurrent tool workers. Serial only for shell, file edits, UI clicks — where order matters.
  • iii.
    Hallucination guard. Streaming output filtered against actual fetched facts before being shown.
  • iv.
    Project-scoped by default. File ops respect the working folder unless you explicitly opt out — predictable blast radius.

SQLite, not the cloud

10Get it running

Install in thirty seconds. Free forever.

One curl pipe and you're done. No card. No login. No daemon. Use the Zen provider for unlimited free runs, or bring your own key.

i.

Run the installer

One line. Drops parth into ~/.local/bin.

ii.

Open a new terminal

Make sure ~/.local/bin is on your PATH (the installer prompts if not).

iii.

Pick "OpenCode Zen" — free

On first run, choose Zen. No payment. No quota. The Parth agent runs free on this provider.

iv.

Run parth in any folder

That folder becomes the working root. Talk to it like an engineer who never sleeps.

install # one-line installer curl -fsSL https://raw.githubusercontent.com/PrajsRamteke/parth-agent/main/scripts/install | bash # open a new terminal, then: parth
